My Personal Movie List

Here is a table below that will show you what each URL endpoint should do while working with a database

Description URL HTTP method Request body data format Response body data format
Get a list of all movies /movies GET None json object holding data about all movies
Get data about a single movie by title /movies/:title GET None json object holding data about a single movie, containing title, description, genre, director,image URL
Get data about a movie genre /genre/:name GET None json object holding data about a movie genre, containing name and description.
Get data about a director /director/:name GET None json object holding data about a director, containing name, bio, birth year, death year.
Add a new user to the registry. /users POST json object holding data about the user to add, appears as:

username: "USER", password: "IlikePIE", email: "USER@mail.com", dateofbirth: "XX/XX/XX"

json object holding data about the user that was add,appears as:

id: X, username: "USER", password: "IlikePIE", email: "USER@mail.com", dateofbirth: "XX/XX/XX"

Update a user's information /users/:username/:password/:email/:date of birth PUT None A text message indicating the user's information was updated.
Add a movie to the user's favorites list /favorites/:username/:title POST None Message indicating the movie title was added to the user's favorite list.
Delete a movie from the user's favorite list /favorites/:username/:title DELETE None Message indicating the movie title was deleted from the user's favorite list.
Remove a user from the registry. /users/:username/:title DELETE None Message indicating the user was removed from the registry.
